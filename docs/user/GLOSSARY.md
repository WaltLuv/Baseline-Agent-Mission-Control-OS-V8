# Glossary

Plain-English definitions of every term used in Mission Control.

- **Mission Control** — Your AI workforce dashboard. The supervision layer where you observe, approve, and steer your AI Employees.
- **Baseline OS** — The brain of the workforce. Holds memory, derives operational truth, and connects skills to employees.
- **AI Employee** — A named, persistent worker with a role, skills, memory, and a track record. Not a chatbot.
- **Skill** — A reusable, attachable capability — e.g. "Send invoice".
- **Workforce Credits** — The unit of usage. Pre-purchased credits that cover compute, model calls, and runtime time.
- **Operator Memory** — Notes and decisions only the operator sees. Private to your workspace.
- **Business Knowledge Base** — Shared knowledge connected from Notion or Obsidian.
- **Knowledge Intelligence** — A deeper memory layer your workforce uses to find relevant past work.
- **Runtime** — An execution engine that runs the work and reports telemetry — Hermes, OpenClaw, or Claude Code.
- **Hermes** — Native Python / JS runtime hook.
- **OpenClaw** — Browser and tool runtime hook.
- **Open Code** — OpenClaw used inside a code editor context.
- **Claude Code** — Reporting layer for repository work.
- **Heartbeat** — A periodic "I am alive" signal from a runtime.
- **Handshake** — The first exchange between a runtime and Mission Control.
- **Trace** — A chronological log of one employee's recent work, memory used, and outcomes.
- **Approval Queue** — Queue of work items awaiting human approval, rejection, or change requests.
- **Skill ROI** — A leaderboard of skills ranked by approved outcomes, time saved, and value created.
- **Proven Capability** — A skill with more than one approved outcome.
- **Demo Mode** — A mode that shows realistic example data so you can explore.
- **Live Mode** — A mode that shows real workspace data.
